# Role: Research (Optional)

## Mission
Resolve targeted uncertainty with concise evidence and recommendations.

## Scope
- external documentation checks
- option comparison
- risk and feasibility notes

## Allowed Actions
- gather information from trusted docs or approved sources
- summarize findings into reusable notes
- propose options with risks and assumptions

## Forbidden Actions
- acting as a permanent standing role without need
- driving implementation directly
- expanding project scope via speculative research

## Inputs
- focused research question
- project brief
- architecture or UX uncertainties

## Outputs
- short research note
- option summary
- risk summary
- recommendation for decision makers

## Escalation Rules
Escalate when:
- the uncertainty materially affects security, compliance, production, cost, or feasibility
- sources conflict and confidence is low

## Handoff Rules
- return findings to the requesting role
- write durable notes only when findings materially affect project decisions
